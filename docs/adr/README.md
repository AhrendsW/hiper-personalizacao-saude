# Architecture Decision Records (ADRs)

ADRs documentam **decisões arquiteturais com consequência durável**. Cada uma segue o template:

- **Contexto** — qual problema motivou a decisão
- **Decisão** — o que foi escolhido
- **Consequências** — o que se ganha e o que se perde

| ADR | Decisão |
|---|---|
| [001](001-lakehouse-vs-warehouse.md) | Lakehouse como backbone de dados |
| [002](002-fhir-canonico.md) | FHIR R4 como modelo canônico de dados clínicos |
| [003](003-ml-classico-para-decisao.md) | ML clássico para decisão clínica e financeira |
| [004](004-llm-camada-de-texto.md) | LLM apenas em camadas de texto e raciocínio |
| [005](005-feature-store.md) | Feature Store dedicada (Feast) |
| [006](006-orquestracao-temporal.md) | Temporal para orquestração de jornadas |
